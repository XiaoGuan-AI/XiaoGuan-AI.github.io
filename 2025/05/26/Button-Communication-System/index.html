🧓 按钮通讯系统：多障人群应急呼救方案（雏形）

📍 作者： Jia’nan Zhao
📅 时间： 2023‑12

本项目旨在为老人、聋哑人、盲人等特殊人群提供一套便捷、可靠的紧急呼救通讯系统。用户通过按键即可向远程接线员发送文本/语音求助信息，支持WiFi远程通信、语音播放、按钮状态反馈等功能。

🧩 适用场景与人群

人群类别 操作方式 系统响应
聋 按键 + 闪灯 灯光提示 + 文本消息发送至 App
哑 按键 + 闪灯 文本发送 + App 提示音
盲 语音提示 + 按键 音频播报反馈 + 语音上传
聋哑 按键 + 闪灯 文本消息发送
聋盲 语音提示 音频消息上传 + 响应播放
聋哑盲 超大按钮 + 闪灯反馈 文本+语音呼救，App 响应震动与语音播报

📲 APP 通信系统架构

📱 App 功能模块

  • ✅ 实时接收文本和语音消息（基于 WebSocket / Firebase 等）
  • 🔔 收到提醒：震动 + 弹窗 + 语音提示
  • 📬 “已收到”按钮 → 通知单片机 LED 闪灯反馈
  • 📞 “接通通话”按钮 → 开启音频通话（WebRTC / SIP 等）

🧠 通讯逻辑

  1. ESP32通过 WiFi 连接服务器
  2. App 与服务器建立持久连接（WebSocket / MQTT）
  3. 短按按钮发送预设文本到 App
  4. 长按录音后发送语音到 App
  5. App 回复“收到”信号 → ESP32控制按钮灯闪烁
  6. App 可远程发起语音通话

💡 系统功能

  1. 🔘 四个大尺寸按钮（可自定义功能,比如: 药店，医院，跑腿，SOS（语音拨号））
  2. 💬 短按发送文本：按键松开即发送预设文本到手机端
  3. 🎙️ 长按录音+发送语音：松开后上传至APP端并播放
  4. 🔄 APP收到回执：按钮闪灯提示已被接收
  5. 📞 接通通话功能：APP端点击按钮后进入语音双向通话
  6. 🌐 远程通信：无需电话卡，基于 WiFi + Web 实现
  7. 🛠️ 后期可自定义每个按钮的发送内容

🖼️ 系统实物示意图

按钮模块 → ESP32 → WiFi → 云服务 → 手机(或PC) App

🔌 硬件组件清单

元件 型号/说明
主控芯片 ESP32（带WiFi）
按钮模块 4个带 LED 的大按钮（可适配行动不便人群）
语音采集模块 MEMS 麦克风模块（I2S/模拟）
音频输出模块 小功放 + 蜂鸣器 / 扬声器
电源模块 移动电源 / Type-C 适配器
其他 杜邦线、电阻、电容、面包板等

🔗 通讯协议设计

目前采用 WebSocket 协议进行通信，后续考虑改用使用 MQTT 更加稳定可靠。

⚙️ APP 端开发建议

功能 技术推荐
通信层 MQTT.js、WebSocket、Firebase Realtime DB
播放音频 Audio() 或 MediaPlayer
语音通话 WebRTC / Agora / PJSIP / Jitsi SDK
框架选择 Flutter（跨平台）、React Native、Kotlin/Swift

📋 系统流程图（App 版本）

graph TD
A[用户按按钮] --> B{短按 or 长按?}
B -->|短按| C[发送文本消息]
B -->|长按| D[录制语音并发送]
C --> E[App 接收并显示]
D --> E
E --> F[App 按“收到”按钮]
F --> G[ESP32 控制 LED 闪烁]
E --> H[App 按“接通”按钮]
H --> I[音频通话建立]

🚀 扩展建议

  • 加入 GPS 模块 → 自动附带地理位置信息
  • 后台网页管理系统 → 可配置按钮文本内容
  • 数据加密传输 → 确保隐私与安全
  • 推送功能 → 未读消息推送提醒
