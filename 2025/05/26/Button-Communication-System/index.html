🧓 按钮通讯系统：多障人群应急呼救方案

📍 作者： Jia’nan Zhao
📅 时间： 2023‑12

本项目旨在为老人、聋哑人、盲人等特殊人群提供一套便捷、可靠的紧急呼救通讯系统。用户通过按键即可向远程接线员发送文本/语音求助信息，支持WiFi远程通信、语音播放、按钮状态反馈等功能。

🧩 场景需求

人群类别 交互方式 系统响应
聋 按键 + 闪灯 灯光提示 + 文本消息发送
哑 按键 + 闪灯 文本发送 + 手机端展示
盲 语音提示 + 按键 音频播报反馈 + 文本或语音上传
聋哑 按键 + 闪灯 文本消息发送
聋盲 语音播报 音频消息上传 + 响应指令播放
哑盲 需外部辅助 （需辅助设备配合）
聋哑盲 4个超大按钮+亮灯提示 文本+语音呼救，网页响应闪灯、音频

💡 系统功能

  1. 🔘 四个大尺寸按钮（可自定义功能,比如: 药店，医院，跑腿，SOS（语音拨号））
  2. 💬 短按发送文本：按键松开即发送预设文本到手机端
  3. 🎙️ 长按录音+发送语音：松开后上传至网页端并播放
  4. 🔄 网页收到回执：按钮闪灯提示已被接收
  5. 📞 接通通话功能：网页端点击按钮后进入语音双向通话
  6. 🌐 远程通信：无需电话卡，基于 WiFi + Web 实现
  7. 🛠️ 后期可自定义每个按钮的发送内容

🖼️ 实物工作环境预览

系统通过 USB 供电，接入多个按钮模块，配合网页进行联动控制

🔌 硬件设计方案

📦 电子元器件清单

元件 型号/说明
主控芯片 ESP32（带WiFi，支持语音采集与GPIO控制）
按钮 4个带LED灯的大按钮（建议直径 ≥ 4cm）
语音采集模块 MEMS麦克风模块（I2S或模拟）
音频播放模块 小型功放板 + 蜂鸣器或微型扬声器
电源模块 5V 电源 / 移动电源 / Type-C接口
辅助模块 杜邦线、面包板、电阻、电容等

🧑‍💻 软件系统设计

📱 前端（网页）

  • 接收并展示文本与语音消息
  • 提供“✅ 收到”按钮 → 硬件端按钮闪灯提示
  • 提供“📞 接通”按钮 → 开启网页音频通话

📡 单片机端（ESP32）

  • 通过 WiFi 连接 WebSocket 服务器
  • 短按：向指定 URL POST 文本
  • 长按：录音后发送音频（base64编码或音频流）
  • 接收网页回应信号 → 控制对应按钮灯闪烁
  • 支持 WebRTC 或 ESP32-Audio-Kit 开源方案进行音频通信

📋 系统工作流程图

graph TD
A[用户按下按钮] --> B{短按 or 长按?}

B -->|长按| D[录制语音]
D --> E[发送语音数据]
C --> F[网页显示]
E --> F
F --> G[用户点击“收到”]
G --> H[按钮LED闪烁提示]
F --> I[用户点击“接通”]
I --> J[开启语音通话]

📎 后续扩展建议

  • 支持 MQTT 云端服务器，便于后端集群处理
  • 添加 GPS 模块 → 自动发送定位地址
  • 后台可在线配置每个按钮功能与文字内容
  • 加入断电语音提示 & 低功耗处理

📦 开发工具推荐

  • 🔧 单片机开发环境：ESP-IDF / Arduino IDE
  • 💻 前端页面：HTML + Vue / React + WebSocket
  • 📶 服务端支持：Node.js / Flask + WebSocket Server
  • 📼 音频编码：ESP32 I2S + Opus 或 PCM → Base64
