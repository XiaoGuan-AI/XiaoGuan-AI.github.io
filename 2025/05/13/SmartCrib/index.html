智能婴儿床控制系统制作流程

1. 系统简介

基于 AIoT（人工智能+物联网）技术，我设计了一个智能多模态婴儿床监控与预警系统。该系统集成了视频监控、人脸识别和环境传感等功能，可实时采集婴儿的图像、声音、温湿度、距离等数据并进行分析。系统能够实时监测婴儿状态，并通过算法预测潜在危险（如踢被子、尿床、发热、哭闹、翻身等），在危险发生前向家长发出报警。通过这种方式，父母即使不在婴儿身旁，也能及时了解婴儿状况、远程查看实时视频，从而提高婴儿看护的安全性和舒适度。

声明： 本项目已在国际知名期刊 PeerJ Computer Science 发表研究论文，并已在国家知识产权局公开专利（申请号：2023116156860，可通过 国家知识产权局官网 查询）。此外，项目已获得多项竞赛奖项，🥉科研竞赛奖项 。

2. 硬件组成与功能描述

ESP32-CAM 模块 + OV2640 摄像头（图1）：ESP32-CAM 是一个集成 Wi-Fi/蓝牙功能的开发板，内置 OV2640 摄像头模块，可拍摄最高1600×1200分辨率的图像。该模块用于婴儿活动的实时视频监控，通过板载 Wi-Fi 热点将视频流推送到家长的移动设备。ESP32-CAM 开发板接受 5V 直流电源并通过稳压芯片输出 3.3V 为 ESP32 芯片和摄像头供电，并通过 USB-串口芯片实现与计算机的通信调试。图1所示的场景中，ESP32-CAM 开发板通过 USB 供电，并通过 Wi-Fi 与模拟的家长手机建立连接，实现婴儿状态的远程查看。

AONI 高清摄像头 + 笔记本电脑（图2）：AONI 高清摄像头通过 USB 接口连接到笔记本电脑，用于“翻身检测”模块。笔记本端运行人脸检测算法，接收来自 AONI 摄像头的实时图像并检测婴儿面部特征。如图2所示，系统通过 Python 脚本调用 OpenCV 等库，对摄像头捕获的每帧图像进行灰度转换和人脸检测；如果连续超过5秒未检测到婴儿面部，则判定婴儿已翻身（可能有窒息风险），系统自动触发蜂鸣器报警。图2中的示例脚本演示了如何抓取婴儿面部图像并保存到本地数据集中，用于训练和优化识别算法。

Arduino UNO 控制模块与传感器组合（图3）：Arduino UNO 开发板作为系统的主控单元，通过 GPIO 引脚采集多种环境传感器的数据，并控制警报输出和数据显示。图3中所示模块组合包括：

  • DHT11 温湿度传感器：用于测量婴儿被褥或环境的温度和湿度。
  • KY-037 声音检测模块：用于检测环境声级并识别婴儿哭闹声的响度。
  • HC-SR04 超声波传感器：用于测量婴儿或物体与传感器之间的距离，判断是否爬出床沿。
  • LCD1602A 显示模块：通过并行接口连接 Arduino，用于实时显示检测到的温度、湿度、声级和距离等数据。
  • 蜂鸣器：连接在 Arduino 的数字输出端口，当系统检测到异常情况（如面部消失、温度过高等）时发出声响警报。
  • 滑动变阻器：通过改变其电阻值来调整系统电流和电压，用于测试和校准环境检测模块。

此外，9V 锂电池经过电源模块将电压稳压为 5V 和 3.3V，为 Arduino 开发板及所有传感器模块供电。Arduino UNO 将采集到的传感器数据通过串口（USB）连接传输至上位机，用于后续数据记录和分析。

3. 系统架构图与连接方式说明

  • ESP32-CAM 与移动设备：ESP32-CAM 开发板通过 USB 接收5V电源，并内置 Wi-Fi 天线，通过热点模式与家长的手机或其他设备建立连接。视频采集后，ESP32-CAM 启动摄像头服务器实时推流，家长通过浏览器访问板卡输出的 IP 地址即可查看监控画面。
  • AONI 摄像头与笔记本：AONI高清摄像头通过 USB 接口连接笔记本电脑，用于采集婴儿床上方的高清视频。笔记本端运行翻身检测算法模块，负责接收视频流并进行人脸检测和状态分析。
  • Arduino UNO 与传感器：Arduino UNO通过电源模块将9V电池电压转换为5V/3.3V，为主控板及传感器供电。各传感器（DHT11、声音传感器、超声波传感器等）的数据线连接至 Arduino 的数字或模拟输入引脚。Arduino 通过程序定时读取这些传感器数据，并通过串口（Serial USB）将采集的数据发送到笔记本。LCD1602A 显示屏通过并行接口连接 Arduino，用于本地实时显示温湿度、声级和距离值。蜂鸣器则连接在 Arduino 的数字输出端口，用于发出报警信号。滑动变阻器置于电路中，用于调节系统电流。
  • 电源和其他模块：系统主要由9V锂电池供电，通过电源模块提供稳定的5V/3.3V输出。开发板的USB接口还可以直接供电和通讯。各模块连接关系可参考论文中环境检测模块的仿真设计图，该设计图详细标示了传感器与 Arduino、LCD、蜂鸣器等设备之间的接线关系，有助于硬件搭建和线路排布。

4. 软件部分说明

摄像头监控功能

ESP32-CAM 开发板运行实时状态监测程序，实现摄像头配置和视频推流。程序初始化 OV2640 摄像头、设置 GPIO 引脚，并连接至指定的 Wi-Fi 热点；连接成功后启动摄像头服务器，将实时视频流通过 HTTP 推送。在开发阶段，通过 Arduino IDE 的串口监视器可以看到板卡输出的本地 IP 地址。用户在手机或电脑浏览器中输入该 IP，即可远程实时查看婴儿床的监控画面。该功能使家长能够随时随地通过移动设备观看婴儿的情况。

数据采集与传输

Arduino UNO 主控程序中，循环读取 DHT11 温湿度传感器、声音传感器(KY-037)、超声波传感器(HC-SR04) 等的数据。采集到的温度、湿度、环境声级和距离值通过 Serial.print 函数实时输出到串口。通过 USB 串口将数据传输至笔记本后，可在上位机端进行记录和分析。与此同时，LCD1602A 显示模块会显示最新的监测数据，动态刷新界面内容。Arduino 程序一般设置固定采样间隔（如每秒更新一次），确保系统能够及时掌握婴儿环境变化，并为后续的多模态数据分析提供支持。

人脸识别与数据记录脚本

在笔记本电脑上，运行基于 Python 的人脸检测与记录脚本。该脚本使用 OpenCV 等库，对 AONI 摄像头采集的连续图像帧进行处理。程序首先将图像转换为灰度图，然后使用预训练的人脸分类器（如 Haar 级联模型）检测人脸位置。若在当前帧中检测到人脸，则在图像上标记面部区域并记录当前时间戳为“正常”状态；若超过 5 秒未检测到任何面部区域，则判定婴儿可能翻身（面部被覆盖）。翻身判定后，程序会触发蜂鸣器报警并记录事件。图2中的示例脚本片段演示了如何使用 getTrainingData 函数连续捕获婴儿面部图像并保存到本地文件夹，用于构建训练集，从而不断优化人脸识别算法的准确性。

5. 数据采集与处理流程

本系统采用多模态数据采集与融合的方式综合分析婴儿状态。具体流程如下：

  • 图像数据：ESP32-CAM 模块对婴儿床进行全时视频监控，实时推送监控画面；AONI 高清摄像头采集婴儿正面图像，供人脸检测和体位识别使用。通过人脸检测算法，系统判断婴儿是否面朝上，若连续一段时间内未检测到面部，则判定翻身风险。
  • 声音数据：声音传感器持续监测环境声级，用于识别婴儿的哭闹声或其他突发声响。当检测到声级超过设定阈值时，可判断婴儿正在哭闹，从而触发相关报警提示。
  • 环境数据：DHT11 温湿度传感器监测被褥和房间的温度湿度，帮助判断婴儿是否踢被子导致过凉，或环境过热对婴儿造成影响。如温湿度超过安全范围，系统会预警婴儿可能发热或过热。
  • 距离数据：超声波传感器测量婴儿与床沿之间的距离，用于检测婴儿是否爬出或靠近床沿从而面临跌落风险。

采集的各种数据由 Arduino UNO 集中处理，并通过软件算法进行分析综合。例如，当 AONI 摄像头持续 5 秒未检测到婴儿面部时，系统认为婴儿翻身；当声音传感器连续检测到高强度哭声时，认为婴儿哭闹；当温度/湿度异常时，认为可能发烧或踢被子。通过对图像和环境传感器数据的多模态融合分析，系统能够在危险发生前及时发出提醒，如检测到翻身风险即触发蜂鸣器，并通过移动设备通知家长，帮助其及时采取措施。

6. 测试与功能验证

系统搭建完成后，需要逐个模块进行功能测试，并进行整体集成验证。在硬件测试方面，对每个传感器进行校准：例如在已知的温湿度环境下测试 DHT11 的读数准确性；将超声波模块指向不同距离的物体，检查测距误差；用标准信号源模拟哭声，测试声音模块的响应灵敏度和阈值；检查 LCD1602 显示的数值是否与实际传感器输出一致。还要验证电源模块和滑动变阻器工作是否正常，保证系统供电稳定。

在软件测试方面，首先验证 ESP32-CAM 的视频流传输稳定性：搭建 Wi-Fi 网络后，观察摄像头服务器推流是否连续可靠。然后测试翻身检测算法：在控制环境下让摄像头捕捉连续面部画面和无面部画面，检查程序是否能正确识别面部消失并触发报警。同时，在 Arduino 程序运行时，通过串口将各传感器输出导出到上位机（例如 Excel）进行统计，对比分析各传感器读数与真实值的一致性。验证期间应确保系统在不同场景下（如日夜光照变化、网络环境变动）均保持稳定工作。

最后进行综合功能验证：模拟多种危险场景（如婴儿翻身、踢被子、哭闹、靠近床沿等），观察系统响应。实验证明，系统能够准确监测到以上多种危险情况并发出及时预警。例如，当婴儿翻身时，AONI 摄像头检测到面部消失后 5 秒内触发蜂鸣器；当环境温度骤升时，DHT11 传感器的读数超过设定阈值时触发报警；哭闹时声音模块触发高声警报。通过这些测试，可以评估系统的稳定性和可靠性，并根据测试结果调整算法阈值和硬件参数以优化性能。

7. 总结与优化建议

本系统成功将视频监控、人脸检测与环境传感器结合，构建了一个婴儿床智能监护与预警方案。系统优势在于利用多模态检测手段，实现了婴儿状态的实时远程监控和多重危险预测，如在婴儿出现踢被、尿床、哭闹、翻身等情况时提前报警，极大减轻了父母的看护负担，提高了婴儿看护的安全性。然而，目前方案仍停留在实验室仿真阶段，尚未在实际家庭环境中进行大规模部署与验证，其稳定性和可行性还有待进一步研究。

未来可在以下方面进行优化：首先，进一步引入机器学习等AI技术，对采集到的多维度数据进行深度学习和智能分析，以提高危险预测的准确率和鲁棒性。其次，可增加更多传感器（例如心率、运动传感器）和交互功能（如语音识别、婴儿情绪分析）来提升系统智能化水平。同时，需要关注数据隐私与伦理问题，确保采集和传输的数据安全可靠。通过不断迭代优化算法和硬件选型，未来该系统有望在托育机构或家庭中广泛应用，为婴儿护理提供更智能、高效的解决方案。
